## 2c. Pass note — Step 1 (Vote-commit experience) ✅

Per the reordered `INSTRUCTIONS.md` Step 1: **(a) commit EBX everywhere a benefactor votes** and **(b) measure races in votes, not EBX**.

**Vote-weight model (new — for your feedback).** Added to `ebx_shared.ts`: `voteWeight(baseVotes, committedEbx) = baseVotes * (1 + committedEbx / EBX_PER_VOTE)` with `EBX_PER_VOTE = 100`. A benefactor casts a base vote (allocated by share when split); **committing EBX adds weight** to that vote but does **not** replace votes as the race metric. The EBX pool (total committed) stays a separate money-side stat. Exposed `voteWeight` + `formatVotes` on `window.EBX`.

**1a — Commit EBX surfaces:**
- **cause.html** phase-1 election widget: new "Commit EBX (adds weight to your vote)" number input in the commit action; `commitVote` now persists `ebx` into `ebx_vote_committed[causeId]` alongside `shares`.

- **cause.html** row subline now leads with the live vote tally (`syncCauseTally` fills "N voters · weighted X%"); EBX shown only as a secondary "· pool N EBX".
